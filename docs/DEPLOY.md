# Deploy: Render (backend) + Vercel (frontend)

Production layout:

```
Browser  →  Vercel (React SPA)  →  Render (FastAPI + Playwright + scheduler)
                                      └── persistent disk: SQLite + screenshots
```

Deploy **backend first**, then **frontend**, then wire CORS.

---

## Prerequisites

- GitHub repo with this project pushed
- [Render](https://render.com) account (Starter plan recommended — see below)
- [Vercel](https://vercel.com) account

### Why Starter on Render?

| Requirement | Free tier | Starter+ |
|-------------|-----------|----------|
| Always-on (scheduler) | Sleeps after ~15 min idle | Always running |
| Persistent disk (DB) | Ephemeral — data lost on redeploy | 1 GB disk at `/data` |
| Playwright RAM | Often too tight | Usually sufficient |

The included [`render.yaml`](../render.yaml) uses **Starter** and a **1 GB disk** at `/data`.

## Part 1 — Backend on Render

### Option A: Blueprint (recommended)

1. Push this repo to GitHub.
2. Render Dashboard → **New** → **Blueprint**.
3. Connect the repo; Render reads [`render.yaml`](../render.yaml).
4. Before first deploy completes, open the **api-saidas-backend** service → **Environment** and set:

   | Variable | Value |
   |----------|--------|
   | `ENCRYPTION_KEY` | Fernet key (generate locally, paste once): `python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"` |
   | `FRONTEND_ORIGIN` | Leave empty for now; set after Vercel deploy |

   `SECRET_KEY` is auto-generated by the blueprint. `CORS_ORIGIN_REGEX` defaults to `https://.*\.vercel\.app` for preview URLs.

5. Wait for deploy; note the public URL, e.g. `https://api-saidas-backend.onrender.com`.
6. Verify: `GET https://YOUR-BACKEND.onrender.com/health` → `"database": "ok"`.

### Option B: Manual web service

1. **New** → **Web Service** → connect repo.
2. **Runtime:** Docker  
3. **Dockerfile path:** `./backend/Dockerfile`  
4. **Root directory:** leave empty (repo root) or set context to `backend` per Render UI.
5. Add a **persistent disk**: mount path `/data`, 1 GB.
6. Set environment variables (same as blueprint in `render.yaml`):

   ```
   DATABASE_URL=sqlite:////data/saidas.db
   SCREENSHOTS_DIR=/data/screenshots
   PLAYWRIGHT_HEADLESS=true
   SCHEDULER_TIMEZONE=America/Sao_Paulo
   SECRET_KEY=<random>
   ENCRYPTION_KEY=<fernet key>
   CORS_ORIGIN_REGEX=https://.*\.vercel\.app
   FRONTEND_ORIGIN=
   ```

7. **Health check path:** `/health`

## Part 2 — Frontend on Vercel

1. Vercel Dashboard → **Add New** → **Project** → import the same GitHub repo.
2. **Root Directory:** `frontend` (important).
3. Framework preset should detect **Vite** (also configured in [`frontend/vercel.json`](../frontend/vercel.json)).
4. **Environment variables** (Production):

   | Name | Value |
   |------|--------|
   | `VITE_API_BASE_URL` | `https://YOUR-BACKEND.onrender.com` (no trailing slash) |

5. Deploy. Note the URL, e.g. `https://api-saidas.vercel.app`.

6. For **Preview** deployments, add the same `VITE_API_BASE_URL` under Preview env if you want previews to hit production API.

---

## Part 3 — Connect CORS (Render)

Back in Render → **api-saidas-backend** → **Environment**:

```
FRONTEND_ORIGIN=https://YOUR-APP.vercel.app
```

Use a comma-separated list for multiple origins:

```
FRONTEND_ORIGIN=https://api-saidas.vercel.app,https://api-saidas-git-main-you.vercel.app
```

`CORS_ORIGIN_REGEX=https://.*\.vercel\.app` already allows other Vercel preview URLs.

Save → Render redeploys automatically.

## Part 4 — Smoke test

1. Open the Vercel URL → register with RA + UNASP password.
2. Settings → confirm profile saves.
3. Dashboard → run a **dry-run** submission.
4. History → confirm status appears.
5. Optional: create a schedule and check `/health` → `scheduler_running: true`.

## Environment reference

### Render (backend)

| Variable | Required | Example |
|----------|----------|---------|
| `SECRET_KEY` | Yes | auto or `secrets.token_urlsafe(48)` |
| `ENCRYPTION_KEY` | Yes | Fernet key |
| `DATABASE_URL` | Yes | `sqlite:////data/saidas.db` |
| `SCREENSHOTS_DIR` | Yes | `/data/screenshots` |
| `FRONTEND_ORIGIN` | Yes (prod) | `https://your-app.vercel.app` |
| `CORS_ORIGIN_REGEX` | Optional | `https://.*\.vercel\.app` |
| `PLAYWRIGHT_HEADLESS` | Yes | `true` |
| `SCHEDULER_TIMEZONE` | Yes | `America/Sao_Paulo` |

### Vercel (frontend)

| Variable | Required | Example |
|----------|----------|---------|
| `VITE_API_BASE_URL` | Yes | `https://api-saidas-backend.onrender.com` |

Rebuild/redeploy frontend after changing `VITE_API_BASE_URL` (baked at build time).

## Updating production

| Change | Redeploy |
|--------|----------|
| Backend code | Render auto-deploy on push (if enabled) |
| Frontend code | Vercel auto-deploy on push |
| New `VITE_API_BASE_URL` | Vercel redeploy required |
| New `FRONTEND_ORIGIN` | Render redeploy (env change) |
| Rotate `ENCRYPTION_KEY` | Users must update password in Settings |

---

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| CORS error in browser | `FRONTEND_ORIGIN` must exactly match Vercel URL (scheme + host, no path) |
| API calls go to localhost | Rebuild Vercel with correct `VITE_API_BASE_URL` |
| Backend crash on start | Set `ENCRYPTION_KEY` (Fernet) and non-default `SECRET_KEY` |
| Scheduler missed jobs | Render free tier sleeps; upgrade to Starter |
| DB empty after redeploy | Attach persistent disk at `/data` or data was on ephemeral FS |
| Playwright timeout | First request after sleep is slow; use Starter always-on |

## Local vs production

| | Local | Production |
|---|-------|------------|
| Frontend | `npm run dev` :5173 | Vercel |
| Backend | `uvicorn` :8000 or Docker | Render Docker |
| API URL | default localhost | `VITE_API_BASE_URL` on Vercel |
| CORS | localhost auto-allowed | `FRONTEND_ORIGIN` on Render |

Local development is unchanged; see [SETUP.md](SETUP.md).
